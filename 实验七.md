## 2. 实验目标与系统特性

本实验摆脱了纯运动学的动画驱动，通过受力分析与数值积分实现了符合物理直觉的柔性体（布料）动态模拟。

### 2.1 核心特性 (Core Features)
- **基于物理的弹性模拟**
  - **胡克定律与阻尼**: 将布料离散化为网格（例如 20x20），基于结构弹簧（Structural Springs）计算相邻质点间的弹性回复力，并引入阻尼力（Damping）以模拟能量耗散，防止系统发散。
  - **防爆处理 (Velocity Clamping)**: 为应对由于显式积分带来的数值爆炸问题，实现了速度截断机制，强制限制质点的最大速度。
- **多重数值积分求解器 (Numerical Integrators)**
  - **显式欧拉 (Explicit Euler)**: 使用当前时刻的力和速度预测下一时刻状态。
  - **半隐式欧拉 (Semi-Implicit / Symplectic Euler)**: 优先更新速度，并使用最新速度更新位置，大幅提升了能量守恒性。
  - **隐式欧拉 (Implicit Euler)**: 采用定点迭代法（Fixed-point Iteration）近似求解未来时刻的受力状态，具有极高的数值稳定性。
- **实时交互控制 (GGUI Interaction)**
  - 借助 `ti.ui.Window` 模块搭建 3D 渲染视窗，并集成控制面板：
    - 支持在三种积分方法之间进行热切换。
    - 提供“暂停 (Pause)”与“重置 (Reset)”布料状态的按钮交互。


## 3. 数值积分与数学基础

系统在离散时间步 $\Delta t$ 内，根据牛顿第二定律 $a = F/m$ 对质点进行状态更新。三种核心积分器的核心数学表达如下：

1. **显式欧拉 (Explicit Euler)**: 
   $$x_{t+1} = x_{t} + v_{t} \Delta t$$
   $$v_{t+1} = v_{t} + a_{t} \Delta t$$
2. **半隐式欧拉 (Semi-Implicit Euler)**:
   $$v_{t+1} = v_{t} + a_{t} \Delta t$$
   $$x_{t+1} = x_{t} + v_{t+1} \Delta t$$
3. **隐式欧拉近似 (Implicit Euler Iteration)**:
   $$v_{t+1} = v_{t} + a_{t+1} \Delta t$$
   $$x_{t+1} = x_{t} + v_{t+1} \Delta t$$
   *(注：由于 $a_{t+1}$ 依赖于 $x_{t+1}$，系统通过多轮迭代逼近该方程的解。)*

## 4. GPU 架构与底层优化策略

为最大化榨取 GPU 的并行计算性能并保证多线程安全，代码在架构上进行了以下深度优化：

1. **多线程并发安全 (Atomic Operations)**: 在计算相邻质点的弹力（作用力与反作用力）时，多个线程可能会同时尝试修改同一个质点的受力缓冲。系统使用了 `ti.atomic_add` 原子操作进行累加，避免了数据写入冲突（Race Condition）。
2. **函数内联与调用开销 (Function Inlining)**: 将力学计算 (`compute_forces_on`) 与速度钳制 (`clamp_velocity`) 声明为 `@ti.func`。Taichi 编译器会在编译期将这些函数直接内联到外层 Kernel 中，彻底消除了 GPU 端的函数调用栈开销。
3. **Kernel 启动优化 (Kernel Merging)**: 为了极致的帧率表现，系统避免了将“受力计算”与“状态更新”拆分为独立的流程。系统将其合并在同一个 `@ti.kernel` 中完成，最小化了每帧循环内 CPU 向 GPU 发起调用的次数。
4. **多阶段同步初始化**: 针对 GPU 严格的读写时序要求，将网格的位置初始化、弹簧拓扑结构生成以及渲染索引构建拆分至独立的 Kernel 中，在 Python 主控域按时序依次调用，确保了物理场初始化的状态同步。
